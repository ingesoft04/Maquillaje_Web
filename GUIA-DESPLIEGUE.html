02 — Antes de empezar

Requisitos del servidor

ℹ️ Necesitas un VPS o servidor con Ubuntu 22.04 / 24.04. Opciones económicas: DigitalOcean ($6/mes), Hetzner (€4/mes), Contabo, o cualquier proveedor con Linux.
💻

Hardware mínimo

1 vCPU · 1 GB RAM · 20 GB SSD

Dominio

Un dominio apuntando a la IP del servidor (registro A en tu proveedor DNS)

🐳

Software

Docker Engine + Docker Compose Plugin (se instalan en el paso 1)

03 — Instalación

Despliegue paso a paso

1

Instalar Docker en el servidor

Conéctate a tu servidor por SSH y ejecuta:

bash — servidor Ubuntu 
# Instalar Docker Engine + Compose Plugin
curl -fsSL https://get.docker.com | sh

# Agregar tu usuario al grupo docker (evitar usar sudo)
sudo usermod -aG docker $USER
newgrp docker

# Verificar instalación
docker --version
docker compose version
2

Subir el proyecto al servidor

Desde tu computador local, transfiere los archivos al servidor:

bash — tu computador local 
# Comprimir el proyecto
tar -czf sena-backend.tar.gz maquillaje-sena-backend/

# Subir al servidor (reemplaza IP con la de tu VPS)
scp sena-backend.tar.gz usuario@TU_IP_SERVIDOR:/home/usuario/

# Conectarte al servidor
ssh usuario@TU_IP_SERVIDOR

# Descomprimir
tar -xzf sena-backend.tar.gz
cd maquillaje-sena-backend
ℹ️ También puedes usar Git: git clone https://github.com/tu-usuario/sena-backend.git
3

Configurar variables de entorno

Crea el archivo .env con tus valores reales:

bash — servidor 
cp .env.example .env
nano .env
.env — editar estos valores 
# Base de datos — usa contraseñas fuertes
DB_USER=sena
DB_PASS=MiContraseñaSegura2025!
DB_NAME=maquillaje_sena

# Redis
REDIS_PASS=OtraContraseñaSegura!

# JWT — genera uno aleatorio:
# openssl rand -base64 48
JWT_SECRET=pega_aqui_el_resultado_de_openssl

# Tu dominio
DOMAIN=tudominio.com
SSL_EMAIL=tu@email.com
FRONTEND_URL=https://tudominio.com
Generar JWT_SECRET seguro 
openssl rand -base64 48
4

Editar el dominio en Nginx

Reemplaza tudominio.com con tu dominio real:

bash 
sed -i 's/tudominio.com/midominio.com/g' nginx/nginx.conf
sed -i 's/tudominio.com/midominio.com/g' docker-compose.prod.yml

También copia el frontend HTML:

bash 
mkdir -p frontend
cp maquillaje-sena.html frontend/index.html
5

Ejecutar el despliegue

bash — despliegue completo con SSL 
# Dar permisos a los scripts
chmod +x deploy.sh manage.sh

# Primer despliegue sin SSL (para el challenge de Let's Encrypt)
./deploy.sh

# Obtener certificado SSL (después del primer deploy)
./deploy.sh --ssl
✔ Si todo funciona, verás tu sitio en https://tudominio.com con candado verde.
04 — Certificado SSL

HTTPS gratuito con Let's Encrypt

⚠️ Antes de obtener el certificado, el DNS de tu dominio debe apuntar a la IP de tu servidor. Verifica con: nslookup tudominio.com
Obtener certificado por primera vez 
./deploy.sh --ssl
Ver estado y renovar manualmente 
./manage.sh ssl-status    # Ver expiración
./manage.sh ssl-renew     # Renovar manualmente
ℹ️ El certificado se renueva automáticamente cada 12 horas mediante el contenedor Certbot. Let's Encrypt emite certificados válidos por 90 días.
05 — Operaciones diarias

Gestionar el servidor

Usa ./manage.sh para todas las operaciones:

Ver todos los comandos disponibles

06 — Referencia rápida

Todos los comandos

ESTADO
./manage.sh statusVer si todos los contenedores están corriendo
./manage.sh healthHealth check completo (API + BD + Redis)
./manage.sh statsCPU, RAM y disco de cada servicio
SERVICIOS
./manage.sh startIniciar todo el stack
./manage.sh stopDetener (sin borrar datos)
./manage.sh restartReiniciar todos los servicios
./manage.sh restart-apiReiniciar solo la API sin tocar BD/Redis
LOGS
./manage.sh logsÚltimas 100 líneas de todos los servicios
./manage.sh logs-apiLogs de la API Node.js
./manage.sh logs-nginxAccesos y errores de Nginx
./manage.sh logs-liveLogs en tiempo real (Ctrl+C para salir)
BASE DE DATOS
./manage.sh backupCrear backup comprimido de PostgreSQL
./manage.sh restore archivo.sql.gzRestaurar desde un backup
./manage.sh db-shellAbrir consola psql interactiva
./manage.sh db-sizeVer tamaño de cada tabla
REDIS
./manage.sh redis-shellConsola redis-cli interactiva
./manage.sh redis-infoEstadísticas de memoria y claves
./manage.sh redis-flush⚠️ Limpiar toda la caché
SSL
./manage.sh ssl-statusVer fecha de expiración del certificado
./manage.sh ssl-renewRenovar certificado manualmente
DEPLOY / ACTUALIZACIÓN
./deploy.shDespliegue completo inicial
./deploy.sh --sslDespliegue + obtener certificado SSL
./deploy.sh --updateActualizar solo la API sin downtime
./deploy.sh --backupHacer backup antes de desplegar

Actualizar la API después de cambios

flujo de actualización 
# 1. Hacer backup antes de cualquier cambio
./deploy.sh --backup

# 2. Subir los nuevos archivos al servidor
scp -r src/ usuario@TU_IP:/home/usuario/maquillaje-sena-backend/

# 3. Actualizar la API (sin bajar BD ni Redis)
./deploy.sh --update

# 4. Verificar que todo funciona
./manage.sh health
